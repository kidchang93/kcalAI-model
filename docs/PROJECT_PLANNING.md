# kCalAIModel Project Planning

## 제품 위치

`kCalAIModel`은 헬스케어 애플리케이션의 식단 분석 기능을 지원하는 AI 추론 서버다. 사용자가 직접 소비하는 메인 제품이 아니라, 앱이 사진 기반 식단 기록을 제공할 수 있도록 음식 분류와 칼로리 설명 생성을 담당한다.

서버의 기획 기준은 모델 시연이 아니라 안정적인 제품 API다. 내부 모델, 외부 LLM, 저장소 구조가 바뀌어도 앱이 소비하는 계약은 최대한 안정적으로 유지해야 한다.

## 시스템 역할

- 음식 이미지 입력을 받아 상위 음식 후보와 신뢰도를 반환한다.
- 선택된 음식명을 기반으로 예상 칼로리와 간단한 영양 참고 정보를 생성한다.
- 향후 식단 이미지 저장, 재분석, 운영 로그, 모델 개선 데이터를 위한 기반을 제공한다.
- 앱이 직접 모델 세부사항을 알지 않아도 되도록 HTTP API 경계를 제공한다.

## 현재 기능 범위

- FastAPI 기반 서버
- `/api/predict` 이미지 분류 엔드포인트
- `/api/gpt-predict` 칼로리 설명 생성 엔드포인트
- `/api/s3/*` 파일 업로드, 삭제, 목록 조회 계열 엔드포인트
- YOLO 기반 음식 이미지 분류
- Hugging Face Inference API를 통한 텍스트 응답 생성
- 로컬 개발용 CORS 허용 설정

## 핵심 API 계약

| 목적 | 메서드 | 경로 | 요청 | 응답 |
| --- | --- | --- | --- | --- |
| 음식 이미지 분류 | `POST` | `/api/predict` | `multipart/form-data`, `file` | `{ predictions: [{ label, score }] }` |
| 칼로리 설명 생성 | `POST` | `/api/gpt-predict` | `{ text, max_tokens }` | `{ response_text }` |
| 파일 업로드 | `POST` | `/api/s3/upload/file` | `multipart/form-data`, `file`, `parent_folder` | 업로드 결과와 객체 URL |

앱이 직접 사용하는 우선 계약은 `/api/predict`, `/api/gpt-predict`다. S3 API는 운영 및 확장 기능으로 분리해서 관리한다.

## 추론 파이프라인

1. 앱이 음식 이미지를 업로드한다.
2. 서버가 이미지 바이트를 RGB 이미지로 변환한다.
3. YOLO 분류 모델이 음식 후보를 예측한다.
4. 서버는 상위 후보를 `{ label, score }` 형태로 정규화해 반환한다.
5. 앱이 선택한 음식명을 서버에 다시 전달한다.
6. 서버는 LLM을 호출해 예상 칼로리, 주요 영양성분, 섭취 참고사항을 짧은 한국어 설명으로 반환한다.

## MVP 기준

- `/api/predict`는 앱이 기대하는 `predictions` 배열을 항상 반환하거나 명확한 오류를 반환한다.
- 예측 결과는 앱에서 바로 표시할 수 있는 음식명과 점수로 정규화된다.
- `/api/gpt-predict`는 한 음식명 기준의 짧은 한국어 설명을 반환한다.
- 모델 파일 경로, Hugging Face 토큰, CORS 설정이 로컬 개발 환경에서 재현 가능하다.
- 서버 내부 예외가 앱의 스키마 파싱 실패로 이어지지 않도록 실패 응답 형식을 정리한다.

## 운영 요구사항

- `HF_TOKEN` 같은 비밀 값은 저장소에 커밋하지 않는다.
- 모델 파일과 학습 산출물은 코드 변경과 구분해 관리한다.
- 추론 시간, 실패율, 업로드 크기, 모델 로딩 오류를 로그로 추적한다.
- 앱에서 사용하는 API 변경은 `k-calAI-RN`과 함께 검증한다.
- CORS 허용 출처는 로컬 개발과 배포 환경을 분리해서 관리한다.

## 제품화 전 보완 사항

- 성공/실패 응답 스키마 일관화
- 음식명 라벨을 사용자 친화적인 한국어 표시명으로 매핑
- 칼로리 설명 프롬프트를 서버에서 템플릿화해 앱 의존도 축소
- 음식량, 1인분 기준, 다중 음식 이미지에 대한 입력 구조 확장
- 이미지 저장 여부와 보관 기간 정책 정리
- 모델 버전, 예측 결과, 응답 시간에 대한 관측 지표 추가

## 비목표

- 이 서버는 헬스케어 앱 전체의 사용자, 인증, 목표 칼로리, 식단 히스토리를 직접 소유하지 않는다.
- 의료적 판단이나 질병별 식단 처방을 제공하지 않는다.
- 모델 실험 결과를 앱 API 계약과 섞어 노출하지 않는다.

## 성공 지표

- 이미지 분류 API의 성공률과 평균 응답 시간
- 상위 후보가 사용자 선택으로 이어지는 비율
- 칼로리 설명 생성 실패율
- 앱 릴리즈 없이 서버 모델을 교체할 수 있는 API 안정성
