## 1. 핵심 의학 사실 (지침 실측)

### 병기별 하루 목표 — `ckd_food_rules.STAGE_TARGETS`

| 영양소 | 비투석 CKD | 혈액투석 | 복막투석 |
|---|---|---|---|
| **단백질** | 0.6–0.8 g/kg (매우 저하 시 0.3–0.5) | **1.2 g/kg** | **1.2–1.3 g/kg** |
| 나트륨 | 2,000 mg 미만(소금 5g) | 3,000 mg | 3,000 mg |
| 인 | 진행 병기서 제한(~800–1,000) | 800–1,000 mg (17 mg/kg) | 800–1,000 mg |
| 칼륨 | 고칼륨혈증 시에만 | 지속 제한 | 지속 제한 |
| 열량 | 30–35 kcal/kg | 35(<60세)/30–35(≥60세) | 30–35(투석액 흡수분 감안) |

⚠️ **단백질 방향이 병기에서 뒤집힌다** — 비투석은 제한, 투석은 증량. 단일 `ckd` 조건은 이를 구분하지 못하므로, 병기별 개인 목표 산출은 **처방에 가까워** 별도 결정이 필요하다(§4).
⚠️ 칼륨·인 제한은 **진행 병기·고칼륨/고인산혈증에서** 적용된다. 초기 CKD에 과도한 제한은 오히려 영양실조를 부른다(KSN1 서문).

### 칼륨 3단계 식품 분류 (KSN1 p107–108 · KSN2)

- **채소**(두 지침 동일): 저(선택)/중(조심)/고(제한) — `VEGETABLE_K_LOW/MID/HIGH`.
  - 고칼륨(제한): 아욱·근대·머위·미나리·부추·쑥·시금치·죽순·취·미역(생)·단호박·늙은호박·쑥갓·고춧잎.
- **과일**: 비투석(KSN1)은 관대, 혈액투석(KSN2)은 엄격(귤·포도가 저→중). 기본은 **투석 기준(엄격)** 인코딩, `on_dialysis=False`면 완화.
  - 고칼륨(제한): 곶감·멜론·바나나·앵두·참외·천도복숭아·토마토·방울토마토·키위.

### 고인 식품 (제한) — `HIGH_PHOSPHORUS_KEYWORDS`

유제품·견과류·잡곡/현미·탄산음료·초콜릿·가공식품(햄/소시지/어묵)·뼈째 생선(멸치/뱅어포). **살코기·생선·달걀 같은 필수 단백질원은 제외**(끊으면 안 되고 인결합제로 관리 — KSN2 p141).

### 고나트륨 식품 (제한) — `HIGH_SODIUM_KEYWORDS`

젓갈·장아찌·자반·라면·어묵·햄·소시지·단무지·절임·장조림·명란·김치류. (국·장류는 한식 주류라 과잉 경고 대신 **저염 조리** 안내로 다룬다.)

### 칼륨 저감 조리법 — `POTASSIUM_REDUCTION_TIPS`

껍질·줄기 제거 → 얇게 썰어 10배 물에 2시간 담금 → 데친 뒤 여러 번 헹굼 / 잡곡밥→흰쌀밥 / 말린 과일 회피(신선·통조림 시럽 제외).

## 2. 왜 이게 필요했나 — 데이터 커버리지 실측 (2026-07-20)

식약처 음식(요리) DB `source='mfds'` 추천 후보 풀의 칼륨·인 커버리지(고유명 dedupe):

- **식사 대분류(아침/점심/저녁)**: 합계 2,197행 중 K/P 있음 1,823행 = **83%**. 국·탕·찌개·나물·찜·전·김치류 100%, 볶음 93%, 밥 92%. → 수치 정렬이 **식사에선 이미 잘 동작**.
- **간식 대분류**: 빵·과자 1%, 음료·차 1% → 칼륨·인 사실상 없음 → CKD 간식 정렬이 NULL 위 랜덤.
- 전체 평균이 "22%"로 낮게 보이는 건 빵(8,032행)·음료(4,638행)가 행 수를 압도하는 착시.

**결론**: 수치 정렬은 **식사에 유효**하고, 간식은 **이름 기반 지침 분류**로 메운다(상보적). 그리고 어느 쪽이든 사용자에게 **수치를 보여준 적이 없다** → "정보를 안 쓴다"는 체감의 진짜 원인.

## 3. 통합 설계 (이번 구현 범위 — 비처방)

### 3-1. 추천에 실측 영양 노출 + 지침 근거 (`recommendation_*`)

`GET /api/recommendations` 계약 변경 (전부 **추가·nullable → 하위호환**, 서버·앱 동시 배포):

- `RecommendationItem` += `sodium_mg` · `potassium_mg` · `phosphorus_mg` · `protein_g` (`float | None`) — DB 실측값 그대로. 원물·미측정 행은 None.
- `RecommendationResponse` += `tips: list[str]` — 사용자 조건 기반 안내(CKD면 칼륨 저감 조리법). 비CKD면 빈 배열.
- 앱은 카드에 수치 칩을, 하단에 tips를 렌더. 값 없으면 숨김.

### 3-2. 고칼륨/고인 후보 배제 (후보 풀) — 이름 + 실측 이중

`_candidate_pool`에서 `low_potassium`/`low_phosphorus` 태그가 있으면 두 축으로 고칼륨·고인 후보를 **하드 배제**한다:

1. **이름 기반**(`POTASSIUM_HIGH_KEYWORDS`·`HIGH_PHOSPHORUS_KEYWORDS` NOT LIKE) — 수치가 없는 간식(바나나·초콜릿·견과)을 잡는다.
2. **실측 상한**(`potassium_mg <= 600` · `phosphorus_mg <= 400`, NULL은 통과) — 정렬만으론 성긴 그룹에서 고칼륨이 새어든다(2026-07-20 로컬 검증: 고구마 804 mg·돼지 간 409 mg 누수 발견 → 상한 도입으로 0건). NULL은 이름 기반이 받는다.

**왜 정렬이 아니라 배제인가**: 고칼륨혈증은 급성 심정지 위험(KSN2)이라 "낮은 순 정렬 후 상위 절반에서 추첨"으론 부족하다. 칼륨(600)은 엄격, 인(400)은 관대 — 인은 만성이고 필수 단백질원을 지켜야 한다. 두 상한은 지침의 절대 컷오프가 아니라 **정책값**이다(§4). 전분질 곡류(고구마·감자·토란)는 KSN 채소/과일 분류 밖이라 `STARCHY_K_HIGH`로 이름 목록에 보강했다.

### 3-3. 기록 경고를 지침 근거로 강화 (`get_record_warnings`)

CKD 조건의 경고를 기존 3키워드(젓갈·장아찌·라면)에서 `ckd_food_rules`의 칼륨/인/나트륨 분류로 확장. 경고 항목에 **어느 영양소인지**를 담아 "칼륨이 높은 편이에요"로 안내(계약: `NutritionWarningItem` += `nutrient: str | None`, 앱 `formatWarning` 분기). *(fast-follow — 이번 커밋 범위 밖일 수 있음)*

### 노출 원칙 (책임)

- **보여주는 것**: 실측 수치 + 지침의 상대 분류(저/중/고) + 저감 조리법 + 고지문. → 사실·상대 안내, **처방 아님**.
- 기존 고지문 유지: `"AI 추정값이며 의학적 조언이 아닙니다."` (CKD 화면엔 "제한 여부·목표량은 병기·검사에 따라 다르니 의료진·영양사와 상담하세요" 보강 권장).

## 4. 보류 — 결정 필요 (처방 근접)

- **병기(비투석/혈액투석/복막투석) 구분 + 개인 단백질 목표(g/kg) 산출.** 단백질 방향 반전을 올바로 반영하려면 필요하나, 개인 목표량 제시는 처방에 근접한다. 데이터모델(건강 프로필에 병기 필드, 비암호화 — 기존 condition/allergen 관례) + 온보딩 + 앱 변경이 따른다. `STAGE_TARGETS`·`PROTEIN_DIRECTION_NOTE`에 근거는 준비돼 있다.
- **절대 임계값**("칼륨 200mg 이상 위험"): 공식 절대 컷오프 부재. 도입 시 신장질환 식품교환표(200mg/교환단위) 근거 명시 + 고지문 강화 필요.
- **추천 배제 상한값**(`POTASSIUM_SERVING_HIGH_MG=600`·`PHOSPHORUS_SERVING_HIGH_MG=400`)은 교환표 200mg/교환단위를 한 끼 제공량으로 환산한 **보수적 정책값**이다. 지침의 공식 컷오프가 아니므로 임상 감수 시 조정한다. 현재는 명백한 고칼륨/고인(고구마·간)을 거르는 안전장치로만 쓰고, 사용자에겐 "위험/금지"가 아니라 추천 후보 선정 기준으로만 반영한다.
