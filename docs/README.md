# kCalAIModel Docs

이 폴더는 `kCalAIModel`을 독립 제품이 아니라 헬스케어 애플리케이션의 식단 분석 기능을 지원하는 추론 서버로 관리하기 위한 운영 문서 모음이다.

## 문서 목록

- `PROJECT_PLANNING.md`
  - 서버의 제품 기획, API 책임, MVP 기준, 제품화 전 보완 사항을 정의한다.
- `SERVICE_POSITIONING.md`
  - 서버의 제품 내 역할, 앱과의 관계, 응답 책임 범위를 정의한다.
- `PROJECT_CONVENTIONS.md`
  - API 설계 원칙, 모델 서비스 규칙, 세션 운영 기준, 변경 관리 규칙을 정의한다.

## 운영 원칙

- `kCalAIModel`과 `kcalai`는 하나의 기능 체인을 이루므로 가능하면 한 세션에서 함께 관리한다.
- API 계약 변경은 서버 내부 리팩터링으로 취급하지 말고 앱 영향 범위를 항상 함께 본다.
- 이 저장소는 메인 제품이 아니라 상위 헬스케어 앱 기능을 지원하는 백엔드라는 위치를 유지한다.
