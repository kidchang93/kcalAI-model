# kCalAIModel Service Positioning

## 제품 관점

`kCalAIModel`은 사용자가 직접 인지하는 메인 서비스가 아니라, 헬스케어 애플리케이션 안의 식단 분석 기능을 지원하는 AI 추론 서버다.

이 프로젝트가 맡는 책임은 다음과 같다.

- 이미지 입력을 받아 음식 분류 및 칼로리 추정에 필요한 결과를 생성한다.
- 앱이 사용할 수 있는 안정적인 API 응답 구조를 제공한다.
- 추론 결과를 제품 플로우에 연결할 수 있도록 예측값, 점수, 부가 정보의 확장 여지를 유지한다.

메인 서비스처럼 보이지 않도록 다음 원칙을 유지한다.

- API 설계는 모델 데모 중심이 아니라 제품 기능 중심으로 본다.
- 서버 응답은 모델 내부 표현보다 앱이 소비하기 쉬운 계약을 우선한다.
- 추론 실험 코드와 제품 API 코드는 가능한 한 구분한다.

## 앱과의 관계

- `kcalai`는 이 서버의 대표 소비자다.
- 서버 엔드포인트, 요청 형식, 응답 스키마가 바뀌면 앱 변경을 동반하는지 먼저 판단한다.
- 서버 단독 최적화보다 앱 사용자 경험 안정성을 우선한다.

## 향후 확장 방향

- 음식 분류 외에 칼로리, 영양소, 설명, 추천 후속 액션을 담을 수 있는 응답 구조를 고려한다.
- 업로드, 추론, 저장, 추적 로그를 분리해 운영 가능하도록 설계한다.
- 모델 교체나 재학습이 있어도 API 계약은 최대한 안정적으로 유지한다.
